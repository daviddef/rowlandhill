# Rowland

An iOS app that identifies postage stamps from a photo — **a whole album page at a time**,
not one stamp at a time — then shows details, catalogue cross-references, and valuations, and
manages your collection.

Named after **Sir Rowland Hill**, who invented the postage stamp and Penny Postage in 1840.

> **Status: early. It builds and runs; it does not yet identify anything.**
> The CoreML embedding model has not been trained, so `StampClassifier.extractEmbedding()`
> returns a placeholder vector, and the backend API is not deployed. What works today is the
> app shell, the Vision detection pipeline, and the page/batch scanning flow.

## Getting started

The `.xcodeproj` is **generated by XcodeGen and is not checked in**, so a fresh clone has no
project file until you generate one:

```bash
brew install xcodegen
xcodegen generate
open Rowland.xcodeproj
```

Target configuration (bundle ID, deployment target, capabilities) lives in `project.yml` —
edit that rather than the project file, or your changes are lost on the next `generate`.

Build and test from the command line:

```bash
xcodebuild -project Rowland.xcodeproj -scheme Rowland \
  -destination 'platform=iOS Simulator,name=iPhone 17' build

xcodebuild -project Rowland.xcodeproj -scheme Rowland \
  -destination 'platform=iOS Simulator,name=iPhone 17' test
```

**Requirements:** Xcode 16+, iOS 17+ target. Signing is not configured (`DEVELOPMENT_TEAM` is
empty in `project.yml`), so simulator builds work and device builds do not.

## How identification works

Not a fixed-class classifier. The model emits a **512-dimensional embedding**, and
identification is a cosine-similarity search against an embedding database — so new stamps
are added to the corpus without retraining.

```
page image
  → normalise EXIF orientation      (Vision reads cgImage and ignores imageOrientation)
  → VNDetectRectanglesRequest       (page-tuned: no observation limit, minimumSize 0.02)
  → deduplicate overlapping boxes   (Vision boxes printed inner frames as well as stamps)
  → reading order                   (rows banded by vertical overlap, then left-to-right)
  → per stamp: embedding → nearest-neighbour search → ScanResult
```

Detected stamps are drawn boxed on the source page so **misses are visible**, and
low-confidence matches arrive **deselected** — a bulk add should never quietly import guesses
into an inventory.

**Condition is user-declared, never inferred.** A photograph cannot see gum, thins, regumming,
or watermarks, and those attributes are frequently the difference between a $5 stamp and a
$5,000 one. Anything claiming to grade condition from a photo is overclaiming.

## Known gaps

- **The embedding model does not exist.** Identification returns placeholder data.
- **The API is not deployed.** Call sites still swallow errors with `try?`, so a failed
  request looks like an empty result.
- **Camera frame capture is unimplemented** — the photo-library path is the working entry point.
- **Rectangle detection finds rectangles, not stamps.** On a real album page it will also box
  mounts, pockets, and page borders. A trained detector is the eventual answer.
- Deep links do not navigate yet; text truncates at accessibility type sizes.
